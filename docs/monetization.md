# Monetization Guide for Mod Publishing

How to earn from your mods ethically — donations, platform programs, and audience building.

## Platforms

### Nexus Mods
- **Program**: Donation Points (DP) — earn points from unique downloads, redeem for cash via PayPal
- **Rate**: Roughly $0.001 per unique download (varies by monthly pool)
- **Best for**: Bethesda games (Skyrim, Fallout), some Unity games (Valheim, Subnautica)
- **Tips**: Opt into DP on every file you upload; it costs nothing

### Patreon / Ko-fi
- **Model**: Direct donations and recurring subscriptions
- **Best for**: Building a following across multiple games
- **Approach**: Offer early access to new mods (time-limited, 1-2 weeks max — never feature-gated)
- **Tiers**: $1-5/month works well for modders; keep it simple

### CurseForge
- **Program**: CurseForge Rewards — ad revenue share based on downloads
- **Best for**: Minecraft mods and modpacks
- **Tips**: Revenue scales with modpack inclusion; get your mod into popular packs

### Thunderstore
- **Model**: Tip-based — users can tip mod authors directly
- **Best for**: Unity game mods (Lethal Company, Valheim, Risk of Rain 2)
- **Growing platform** with an active community

### Steam Workshop
- **No direct monetization**, but drives traffic to your Patreon/Ko-fi
- Link donation pages in your mod description
- Large built-in audience for discovery

## Ethics

- **Never paywall features** — all mod functionality must be free for everyone
- **Donations and tips only** — early access for patrons is OK for a limited window (1-2 weeks)
- Always credit dependencies and inspirations in your mod description
- Respect game developers' modding policies — check EULAs before monetizing
- If a game's TOS prohibits monetized mods, honor that

## Workshop SEO & Presentation

### Title
- Clear and descriptive — include the key feature name
- Bad: "My Cool Mod v3" / Good: "Auto-Sort Inventory — Configurable Storage Manager"

### Description Structure
1. **Hook line** — one sentence explaining the value
2. **Feature list** — bullet points of what it does
3. **Installation instructions** — step by step
4. **Configuration** — what settings are available
5. **Compatibility** — known conflicts and tested-with lists
6. **Credits** — dependencies, inspirations, contributors

### Tags & Images
- Use all relevant platform tags; don't over-tag with unrelated categories
- Preview images: in-game screenshots showing the mod in action
- Image sizes: 512x512 for Steam Workshop, 1280x720 for Nexus Mods
- Before/after comparisons work well for visual mods

### Update Cadence
- Regular updates signal an active mod and boost search visibility
- Even small fixes count — platforms reward recent activity in search rankings

## Building an Audience

### Branding
- Use a consistent username across all platforms (Nexus, CurseForge, GitHub, Discord)
- Cross-promote between games — players of one game often play others in the genre

### Community Engagement
- Respond to bug reports within 24-48 hours — responsiveness builds trust
- Post update changelogs with every release — visible activity retains subscribers
- Share development progress on relevant subreddits and Discord servers
- Pin a FAQ or known-issues list to reduce repeat questions

### Portfolio
- Create a personal mod page or GitHub profile linking all your mods
- A central hub makes it easy for fans of one mod to discover your others
- Include download counts and platform links as social proof

## Licensing

### Recommended Licenses
- **MIT**: Maximum adoption, minimal friction — best for open mods with donation support
- **LGPL**: Allows use as a library without forcing derivative works open
- Avoid **GPL** for mods — it complicates compatibility with proprietary game code
- Avoid **proprietary/no-license** — it scares away users and other modders

### Best Practices
- Include a `LICENSE` file in every mod repository
- Credit attribution for any borrowed code, even if the license doesn't require it
- If you use someone else's mod as a dependency, check their license first

## Revenue Expectations

| Source | Typical Range | Notes |
|-|-|-|
| Most mods | $0-50/month | Donations from a small user base |
| Nexus DP (10k+ downloads) | $50-200/month | Scales with unique download volume |
| CurseForge Rewards | Varies widely | Significant for popular MC modpacks |
| Patreon | $20-500/month | Depends on audience size and tier structure |

- The key is **volume across multiple games and mods**, not banking on one hit
- Treat donation income as a bonus, not a salary — it fluctuates month to month
- Diversify across platforms to smooth out revenue

## Quality Signals That Drive Downloads

- **Frequent, stable updates** after game patches — be the first to update
- **Clear changelogs** with every version — users want to know what changed
- **Fast bug response** — aim for 24-48h first response on reports
- **Compatibility lists** — document which mods work together and which conflict
- **Clean, readable code** that other modders can learn from and extend
- **Config files** — let users customize behavior without editing code
